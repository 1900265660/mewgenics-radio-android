### Phase 2：在线 + 按需缓存架构

已实现核心架构：

- `RemoteRadioManifest.kt`
  - 解析远程 `manifest.json`
  - 包含 `id`、`category`、`relativePath`、`url`、`bytes`、`sha256`、`durationMs`、`codec`、`bitrateKbps`
- `RadioCacheManager.kt`
  - 缓存目录：`context.filesDir/radio-cache/`
  - SHA-256 校验
  - 缓存大小统计
  - 清理缓存
  - 下载并校验单曲
- `RadioAssetResolver.kt`
  - 优先级：有效缓存 > 远程 URL > 内置 assets
- `RadioPlayer.kt`
  - 使用通用 URI 播放，不再局限于 `asset:///`
- `RadioViewModel.kt`
  - 加载 bundled/remote manifest
  - 自动缓存远程 track
  - 提供缓存 songs / 完整 radio / 清理缓存能力
- UI
  - 显示 `Bundled` / `Remote` / `Cached`
  - 显示缓存大小
  - 显示音质标签
  - 支持缓存 songs、缓存完整 radio、清理缓存

已新增本地服务器工具：

```text
tools/serve-radio-assets.py
```

功能：

- 静态服务 `dist/radio-assets/128kbps`
- 支持 HTTP Range
- 返回 `206 Partial Content`
- `.opus` MIME：`audio/ogg`
- 支持 `HEAD`
- 带 CORS
- 有路径保护

已新增 `onlineDebug` 构建类型：

- Application ID：`com.local.mewgenicsradio.online`
- 使用 debug 签名
- 允许本地 cleartext HTTP
- 不打包 `app/src/debug/assets/radio`
- 用于在线测试的小体积 APK

Release 构建默认不允许 cleartext HTTP；本地 HTTP release 测试需要显式传：

```powershell
-PALLOW_CLEARTEXT=true
```

